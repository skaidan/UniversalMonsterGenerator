# Universal Monster Generator - Product Manager Instructions

## Visión del Producto
Crear un sistema eficiente y escalable para generar criaturas/monstruos únicas combinando múltiples bases de características. El sistema debe ser:
- **Modular**: Fácil agregar nuevas bases y características
- **Determinista**: Cada combinación produce resultados predecibles
- **Extensible**: Soporte para rareza, modificadores, escala

## Arquitectura Actual

### Capas del Sistema
```
┌─────────────────────────────────────┐
│   dynamic_loader.py (Orquestador)   │
│   - Selecciona clases base aleatoriamente
│   - Extrae stats de Base1
│   - Recolecta abilities de Base2 & Base3
└─────────────────────────────────────┘
                    │
      ┌─────────────┼─────────────┐
      ▼             ▼             ▼
   bases1/      bases2/       bases3/
   (Stats)      (Habilidades) (Habilidades)
```

### Responsabilidades por Directorio

| Directorio  | Responsabilidad | Stats | Abilities |
|-------------|-----------------|-------|-----------|
| **bases1/** | Estadísticas base | ✅ Define | Contribuye |
| **bases2/** | Mecánicas/poderes | ❌ No | Contribuye |
| **bases3/** | Habilidades especiales | ❌ No | Contribuye |

## Decisiones Arquitectónicas Clave

### ¿Por qué separar stats en Base1?
- Las estadísticas base determinan el "poder general" de la criatura
- Base1 es el recurso más valioso y debe ser seleccionado cuidadosamente
- Permite balanceo justo: todas las Base1 tienen poder equivalente

### ¿Por qué recolectar abilities de las 3?
- Crea combinaciones emergentes e impredecibles
- Aumenta la replayability
- Evita que una rama (Base2 o Base3) domina sin consecuencias

### ¿Dataclass para GlobalCreatureBaseClass?
- Serialización automática a dict/JSON
- Validación en `__post_init__`
- Type safety nativa de Python
- Normalización de abilities deduplicadas

## Notas de Desarrollo

- El `load_classes_from_dir()` carga todas las clases desde archivos .py
- Cada clase en `bases1/` DEBE tener `DEFAULT_STATS`
- Las abilities se normalizan y desduplican automáticamente
- El MRO (Method Resolution Order) de Python determina el orden de ejecución
